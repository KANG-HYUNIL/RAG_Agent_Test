### 260409_제8차 실험 (최종 E2E 검증) 진행 순서

#### 전제 (Inference Environment Constraints)

- **배포 환경 모사**: 8차 실험은 Docker 컨테이너로 패키징된 실제 인퍼런스 서버를 대상으로 수행함.
- **카테고리 필터 활성화 (Category Filter Enabled)**: `inference_test.py`에서 전달하는 `[Category]` 태그를 서버가 파싱하여 리트리버의 `metadata_filter`로 적용함.
- **Answer 필드 영구 제거 (Clean RAG)**: 7차 실험에서 확인된 ICL 편향(정답 유도 노이즈)을 방지하기 위해, 데이터베이스와 프롬프트 예시에서 `answer` 필드를 완전히 배제함.
- **자동화 관리**: `test/inference_test.py`를 사용하여 컨테이너 기반의 전체 로그 수집 및 결과 집계를 자동화함.

#### 8차 핵심 설계 개념 (Real-world Simulation)

##### 1. 실제 질의 파싱 검증 (`_parse_query_text`)
- 실제 사용자 입력 포맷인 `"[Category] 질문내용? ① 선지1 ..."` 형태를 서버에서 파싱.
- 카테고리 정보를 추출하여 검색 도메인을 제한하고, 질문과 선택지를 분리하여 `dense_qpc` 리랭킹에 반영.

##### 2. Clean Prompt 환경에서의 최종 성능 측정
- 7차 실험까지 존재했던 예시 문항의 `answer` 필드를 제거하여, LLM이 실제 리트리빙된 컨텍스트에만 의존하여 답변하도록 설정.
- 정답 정보가 누락된 상태에서 리랭킹 보정 로직이 얼마나 정확도를 방어해주는지가 핵심 지표임.

### [종합 분석 및 인사이트]

#### 1. 주요 발견 (Findings)

    *   법령명 매칭에만 집중했을 때 Law 부문은 소폭 상승(54.35%)했으나, 형법 부문에서 오히려 성능이 하락(27.59%)했습니다. 


#### 2. 배포를 위한 최종 기술적 패턴

*   **Clean Query의 힘**: 에이전트 내부에서 질문과 선지를 명확히 분리하여 `build_query_text`에 전달하는 로직이 성공적으로 작동했습니다.
*   **카테고리 필터 + 리랭킹 시너지**: 카테고리 필터로 검색 범위를 1차 압축한 후, `statute + polarity + choice` 리랭킹이 2차로 조문의 정합성을 보정하는 이중 방어 체계를 구축했습니다.
*   **Latency 최적화**: Docker 환경에서 평균 1.3초 내외의 응답 속도를 유지하며 실시간 서비스 가능 수준을 확보했습니다.

#### 3. 최종 결론

8차 실험을 통해 **Category Filter와 복합 리랭킹 조합이 실전에서 가장 강력한 대안**임을 입증했습니다. 이는 단순히 분야를 좁히는 것을 넘어, 법률 문장의 미세한 차이를 리랭킹 로직이 잡아내어 54% 이상의 신뢰도를 확보했음을 의미합니다. 특히 정답 정보가 없는 순수 RAG 환경(Clean RAG)에서의 성과라는 점이 중요합니다.

> [!IMPORTANT]
> **최종 선정 전략: 8-3 (Full Complex Pipeline)**
> 카테고리 필터가 적용된 상태에서도 리랭킹 보정이 베이스라인 대비 정확도를 유의미하게 끌어올렸으며, 법리적 모호성을 극성 및 키워드 보정으로 해결하는 시너지가 확인되었습니다.
