### 260409_제7차 실험 진행 순서 및 결과 보고

#### 전제

- 결과 저장 방식, 파싱 항목, 실행 방식은 1차~6차 실험과 동일
- 6차 실험 결과, 최고 기준선은 `stage6__baseline_recheck` (55.21%) 및 `stage6__dense_rerank_polarity` (55.98%) 임.
- 5차(Query Reformulation)와 6차(Post-Retrieval Correction) 결과를 분석한 결과, 다음 병목 현상들을 확인:
  - **법령명 혼동 (Statute Mismatch)**: '민법' 문제에 '민사소송법' 판례가 검색되거나, '형사법' 카테고리의 현저히 낮은 성능(37%).
  - **선택지 간의 미세한 차이**: 질문 본문은 포괄적이나 정답은 선택지에 등장하는 세부 법률명사(공소시효 vs 취득시효 등)에 의해 결정됨.
  - **극성 혼동 (Polarity Mismatch)**: 긍정/부정(옳은/틀린) 문제에 반대 방향의 해설이 검색됨.
- 따라서 7차 실험의 핵심은 최고 기준선인 `dense_qpc` Retrieval Backbone을 유지하면서, **결정론적 단서(법령명, 극성, 선택지 고유 단어)를 활용한 보수적인 후단 보정(Conservative Deterministic Reranking)**을 수행하는 것.

#### 7차 핵심 설계 개념 (Conservative)

##### 1. Retrieval Backbone 고정
- Candidate generation은 기존 최고 기준선(`dense + question_plus_choices`)을 유지함.
- Hard filter(강제 제외)나 Hard pruning은 베이스라인을 무너뜨릴 위험(오탐 시 정답 배제)이 크므로 배제.

##### 2. Statute Mismatch Soft Penalty (법령명 보정)
- Query와 Candidate 문서에서 일반 법개념(불법행위 등)을 제외한 **명시적 법령명 Whitelist**(민법, 형사소송법, 공직선거법 등)만 추출.
- 양쪽의 법령명이 아예 다르면(Mismatch) 중간 Penalty, 쿼리에만 명시되어 있고 문서엔 없으면 약한 Penalty 적용.
- 목적: 동일 카테고리 내의 Near-miss(아차상) 방지.

##### 3. Soft Polarity Penalty (극성 보정)
- Query와 Candidate의 Polarity가 명확히 반대일 경우(opposite polarity) Soft Penalty 부과.
- OX나 바르게 연결형과 같이 Polarity가 혼재된 템플릿의 경우 Penalty 생략(보수적 접근).
- 목적: Prefix 태그가 아닌 Reranking 신호를 이용해 부정/긍정 방향 오염을 완화.

##### 4. Choice-Unique Legal Keyword Weak Bonus (선택지 고유명사 보정)
- 질문 본문에는 없고 4개의 선택지(A~D)에만 등장하는 **고유 법률 명사**를 추출.
- 후보 문서와 해당 단어들이 Overlap될 경우 약한 보너스(Weak Bonus) 부여.
- 일반 명사 Overlap으로 인한 Distractor(오답 유도) 유입을 막기 위해 가중치를 제한(Capped)하여 적용.
- 목적: 질문이 포괄적일 때, 정답을 가르는 선택지 특화 단어를 반영.

##### 5. Natural Polarity Suffix (보조 Ablation)
- 기존의 불완전했던 Prefix Polarity(`[긍정방향]`, `[부정방향]`) 대신, `"이것은 옳은 설명이다."`, `"이것은 틀린 설명이다."`와 같은 완성된 자연문장형 Suffix를 Query 끝에 주입하는 방식을 보조 실험으로 검증.

#### 목적

- 추가적인 LLM API 호출(시간 및 비용 문제) 없이 `korean_tokenizer.py`의 결정론적 규칙(Regex, Set)만으로 벤치마크 점수 개선.
- Hard Rule로 인한 역효과를 피해 Soft Penalty/Bonus가 예측 가능한 범위 내에서 작동하는지 개념 검증.
- 저조한 `Criminal Law` 부문을 끌어올릴 수 있는지 확인.

- **LLM 추론 편향**: gpt-4o-mini 모델이 제공된 예시의 '정답' 레이블에 매몰되어, 검색된 컨텍스트보다 과거 패턴을 우선시하는 경향이 포착되었습니다.
- **결론**: 실전 인퍼런스 환경과 동일하게 **정답 필드를 제거한 Clean RAG 상태**에서 다시 검증이 필요함.

7-2차 실험은 7-1차의 ICL 노이즈를 제거하고, 리랭킹 알고리즘의 순수 유효성을 검증하였습니다.

#### 7-2차 실험 분석 및 인사이트 (중요 합산)

**1. 주요 발견 (Findings)**
- **Answer Removal의 승리**: 정답 필드를 제거한 것만으로 정확도가 51%대에서 55.21%로 **약 4%p 반등**했습니다.
- **형법(Criminal Law)의 유효성**: `statute` 리랭킹이 형법 분야에서 최고수치(**34.48%**)를 기록하며, 법령명 매칭이 형사/특별법 문제에서 검색 품질 개선에 기여함을 입증했습니다.
- **일반법(Law)의 트레이드오프**: 명확한 법령명이 없는 민법/일반법 문항에서는 리랭킹 감점이 오히려 노이즈로 작용하여 점수가 소폭 하락하는 현상이 관찰되었습니다.

**2. 7차 실험 최종 결론**
- **Backbone의 중요성**: 리랭킹 알고리즘보다 `dense_qpc` Retrieval Backbone이 성능의 기저를 결정합니다.
- **Post-Retrieval의 방향성**: 결정론적 리랭킹은 특정 카테고리(형법)에 강점이 있으나, 전체 범용성을 위해서는 더 정교한 가중치 튜닝이 필요합니다.
- **8차 실험으로의 연결**: 인퍼런스 시 카테고리 정보가 없는 환경에서의 실전 성능 측정이 최종 배포 전략 수립의 핵심이 될 것입니다.
