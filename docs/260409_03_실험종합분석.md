# RAG Agent 프로젝트: 법률 Q&A 성능 향상 종합 실험 분석 보고서

## 1. 개요 및 실험 로드맵
본 프로젝트는 초기 30%대의 정확도에서 시작하여, 8차에 걸친 정밀 실험을 통해 최종 **54.05%**의 실전 정확도를 확보함. 각 단계는 검색(Retrieval), 보정(Reranking), 추론(Generation)의 세 가지 축을 최적화하는 과정으로 진행됨.

---

## 2. 실험 단계별 요약 및 핵심 인사이트

### [Stage 1-3] 기초 설계 및 모델 선정
*   **핵심 전략**: Baseline 수립 및 임베딩 모델 비교 검증.
*   **대표 인사이트**:
    1.  **선택지 인지 쿼리**: 질문에 선택지(A, B, C, D) 텍스트를 포함하여 임베딩하는 방식(`question_plus_choices`)이 단순 질문 대비 **+3.86%p**의 압도적인 성능 향상을 보임.
    2.  **모델 최적화**: 다국어 모델(`multilingual-e5-large`)이 한국어 법률 용어의 의미적 유사성을 가장 잘 포착함을 확인.

### [Stage 4] 검색 정제 (Noise Filtering)
*   **핵심 전략**: 유사도 임계값(`score_threshold`) 도입.
*   **대표 인사이트**:
    - **임계값의 균형**: `threshold=0.35` 설정 시, 정답과 무관한 노이즈 문서는 배제하면서도 정답 근거를 포함한 문서는 안정적으로 리트리빙하는 균형점 발견.

### [Stage 5-6] 하이브리드 검색 및 도메인 특화
*   **핵심 전략**: BM25(키워드) + Dense(의미) 결합 및 카테고리별 가중치 조절.
*   **대표 인사이트**:
    - **형태소 분석의 중요성**: Kiwi 형태소 분석기를 도입하여 조사와 어미를 제거하고 법률 용어(NNG) 중심으로 BM25를 수행함으로써 키워드 매칭의 정합성 대폭 개선.

### [Stage 7] 결정론적 리랭킹 (Reranking Breakthrough)
*   **핵심 전략**: 법령명(Statute), 극성(Polarity), 선택지 고유 키워드 매칭 보정.
*   **대표 인사이트**:
    1.  **극성(Polarity) 오류 보정**: "않은 것은?"과 같은 부정형 질문에 대해 긍정형 컨텍스트가 상단에 오는 고질적인 문제를 리랭킹 단계에서 감점 처리하여 해결.
    2.  **법령명 매칭**: 질문에서 언급된 법령(예: 민법)과 문서의 메타데이터가 일치할 경우 가점을 부여함으로써 검색의 도메인 정확도 확보.

### [Stage 8] 최종 실전 검증 (E2E Validation)
*   **핵심 전략**: Docker 배포 환경 모사 및 프롬프트 클렌징(Clean RAG).
*   **대표 인사이트**:
    - **Clean RAG의 성능 방어**: 예시 문항에서 정답 정보(`answer` 필드)를 완전히 제거하여 LLM의 '답변 리킹' 가능성을 차단했음에도, 리랭킹 보정 덕분에 **54.05%**의 견고한 정확도 유지.

---

## 3. 최종 최고 성과 전략 (Strategy 8-3)

- **검색**: 카테고리 필터링 + BM25 & Dense Hybrid Search
- **보정**: 법령명(0.85) / 극성(0.95) 페널티 및 선택지 키워드(1.02) 보너스 적용
- **추론**: Clean Prompt Strategy (예시의 정답 정보 배제)

---

## 4. 종합 결론
본 프로젝트를 통해 법률 도메인 RAG 시스템에서 **단순 유사도 검색의 한계**를 명확히 진단하고, 이를 **결정론적 리랭킹(Deterministic Reranking)**과 **실전적 프롬프트 설계**로 보완함으로써 실전 배치 가능한 수준의 신뢰성을 확보함.

> [!TIP]
> **향후 과제**: 형법(Criminal Law) 영역의 고질적인 저성능(31%) 해결을 위해, 법리적 선후관계를 이해할 수 있는 ‘법률 지식 그래프’ 기반의 Reasoner 에이전트 도입을 제안함.
